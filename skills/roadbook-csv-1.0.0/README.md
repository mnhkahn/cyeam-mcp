# Roadbook CSV：一行 CSV 秒变高德地图路书

> 开源 Claude Skill，让 AI 帮你做旅行规划。CSV 丢进去，地图路书自动出。

---

## 痛点：做旅行路书太麻烦了

每次出门旅行，最烦的不是订机票，而是**做路书**。

- 第一天住哪、第二天去哪、中午吃什么、晚上住哪……
- 打开地图一个个搜地点、收藏、再串成路线
- Excel 表格里写了一大堆，最后还得手动导到地图 App
- 多人出行还要分享链接，别人打开全是乱码

我之前去淄博玩，计划了三天行程，光是整理地点+做地图就花了两个小时。回来一想，这事儿不就是「整理 CSV → 生成地图」吗？为什么不能自动化？

于是写了这个 Claude Skill：**Roadbook CSV**。

---

## 核心思路：CSV 是桥梁，AI 是翻译

这个 Skill 的思路特别简单：

```
你的旅行计划（自然语言）
    ↓ AI 整理
CSV 文本（名称,地址,类型,日期,备注）
    ↓ API 提交
高德地图路书（带分享链接 + 二维码）
```

你只需要用大白话告诉 Claude 你的行程，比如：

> "第一天住淄博酒店，第二天早上去八大局吃早点，然后参观陶瓷琉璃馆，晚上吃烧烤。"

Claude 会自动整理成标准 CSV，调用 API，**秒回你一个高德地图链接**，打开就是完整的路书，所有地点都在地图上标好了，还能按天切换。

---

## 使用效果

### 自动多日串联
CSV 里写 `Day1`、`Day2`，地图自动按天分页。第一天最后一个住宿地点，会自动变成第二天的起点。

### 酒店跨天衔接
如果前一天晚上住酒店，第二天地图会直接从这个酒店出发，不用你手动设置起点。

### 类型自动区分
景点、餐饮、住宿、起点，不同类型在地图上用不同图标，一目了然。

### 一键分享
返回的结果包含：
- **分享链接** → 微信直接转发给朋友
- **二维码** → 截图发朋友圈
- **分享 ID** → 随时可以修改覆盖

## 安装 & 使用

### 第一步：安装 Skill

```bash
clawhub install roadbook-csv-1-0-0
```

或者手动把 skill 文件夹放进 `~/.claude/skills/` 目录。唯一依赖是系统自带的 `curl`。

### 第二步：申请 API Key（免费）

1. 打开 [Cyeam 开放平台](https://cyeam-open-main-d02895c.zuplo.site/api/~endpoints)
2. 右上角 Sign In 登录/注册
3. 进 Developer Portal → Create API Key
4. 设置环境变量：
   ```bash
   export CYEAM_API_KEY="你的key"
   ```

### 第三步：告诉 Claude 你的行程

直接丢计划给 Claude，比如：

> "五一去淄博，第一天住张店区，第二天逛八大局、陶瓷琉璃馆、海岱楼，晚上吃烧烤。"

Claude 会自动：
1. 整理成 5 列 CSV
2. 调用 API 生成路书
3. 返回分享链接

### 第四步：打开链接，看地图

点链接就是高德地图路书页面，所有地点自动地理编码，路线、距离、时间全都有。

## CSV 格式（超简单）

```csv
名称,地址,类型,日期,备注
淄博酒店,淄博市张店区华光路,住宿,Day1,入住
八大局便民市场,淄博市张店区共青团东路,餐饮,Day2,早市逛吃
中国陶瓷琉璃馆,淄博市张店区华光路320号,景点,Day2,国家级免费
海岱楼钟书阁,淄博市张店区齐盛湖公园内,景点,Day2,夜景打卡
牧羊村烧烤,淄博市张店区林泽街,餐饮,Day2,晚餐
```

就 5 列，逗号分隔，没有学习成本。

## 为什么开源？

这个 Skill 本身没有任何商业目的，纯粹是解决自己做路书的痛点。代码就是一个 Markdown 文件，没有任何黑盒逻辑，所有 API 调用都是明文 curl，数据存在 Redis 30 天后自动过期。

如果你也烦做路书，欢迎安装试试。

```bash
clawhub install roadbook-csv-1-0-0
```
